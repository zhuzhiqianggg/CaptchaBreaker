## 2. 核心组件

### 2.1 PaddleOCR 引擎

#### 模型架构
PaddleOCR 使用两阶段检测+识别架构：

```
输入图片 → 文本检测(Detector) → 裁剪文本区域 → 文本识别(Recognizer) → 输出文本
```

#### 使用的模型

| 模型 | 用途 | 类型 | 大小 |
|------|------|------|------|
| `PP-OCRv5_server_det` | 文本区域检测 | 检测器 | ~88MB |
| `PP-OCRv5_server_rec` | 中文文本识别 | 识别器 | ~12MB |
| `en_PP-OCRv5_mobile_rec` | 英文文本识别 | 识别器 | ~10MB |
| `PP-LCNet_x1_0_textline_ori` | 文本行方向分类 | 分类器 | ~5MB |

#### 模型存储位置
```
~/.paddlex/official_models/
├── PP-OCRv5_server_det/
├── PP-OCRv5_server_rec/
├── en_PP-OCRv5_mobile_rec/
├── PP-LCNet_x1_0_textline_ori/
├── PP-LCNet_x1_0_doc_ori/
└── UVDoc/
```

### 2.2 图像预处理管道

#### v5.0 当前策略

```
原始图片 → RGB转换 → 智能上采样 → 灰度化 → 对比度增强(1.5x) → 输出
```

| 步骤 | 作用 | 参数 |
|------|------|------|
| RGB转换 | 统一图片格式 | - |
| 智能上采样 | 小图片放大到200x60以上 | 自适应缩放 |
| 灰度化 | 消除颜色干扰 | PIL `.convert("L")` |
| 对比度增强 | 使字符与背景差异更明显 | 增强因子: 1.5 |

#### 预处理演进历史

| 版本 | 策略 | 准确率 | 问题 |
|------|------|--------|------|
| v4.0 | 过度预处理(二值化+连通域+形态学) | 0% | 破坏了文字信息 |
| v5.0 | 轻量预处理(上采样+对比度增强) | ~60% | 平衡了效果与稳定性 |

## 3. 数据流

### 3.1 上传识别流程

```
客户端
  │
  ├─ 1. POST /ocr/upload (multipart/form-data)
  │   ├─ file: captcha.png
  │   └─ language: "en"
  │
  ▼
FastAPI 服务器
  │
  ├─ 2. 保存上传文件到 temp/
  │
  ├─ 3. 预处理图像
  │   ├─ RGB转换
  │   ├─ 上采样 (if small)
  │   └─ 对比度增强
  │
  ├─ 4. 调用 PaddleOCR.ocr()
  │   ├─ 文本检测 → 找到文字区域
  │   └─ 文本识别 → 识别文字内容
  │
  ├─ 5. 解析结果
  │   ├─ 提取文本、置信度、边界框
  │   └─ 合并文本，清理空格
  │
  ├─ 6. 清理临时文件
  │
  └─ 7. 返回 JSON Response
         ├─ success: true
         ├─ full_text: "A7k9"
         ├─ texts: [...]
         └─ preprocessing_applied: [...]
```

### 3.2 内存管理

| 资源 | 管理方式 |
|------|----------|
| 上传文件 | 保存到 `temp/` 目录，处理完成后删除 |
| PaddleOCR 模型 | 启动时全局加载，常驻内存 |
| 处理中间图片 | 临时保存后立即删除 |
| API 请求 | 异步处理，自动释放 |

## 4. 关键技术实现

### 4.1 智能上采样

```python
def preprocess_image(image):
    width, height = image.size
    if width < 200 or height < 60:
        scale = max(200/width, 60/height)
        new_w = int(width * scale)
        new_h = int(height * scale)
        image = image.resize((new_w, new_h), Image.LANCZOS)
```

**原理**: 验证码图片通常很小(~100x40)，直接识别时字符像素太少。放大后每个字符有更多像素，OCR 模型能更准确地识别。

### 4.2 对比度增强

```python
img_gray = image.convert("L")
enhancer = ImageEnhance.Contrast(img_gray)
img_enhanced = enhancer.enhance(1.5)
```

**原理**: 增强字符与背景的对比度差异，使 OCR 检测器更容易区分文字区域和背景区域。

### 4.3 PaddleOCR 结果解析

```python
# PaddleOCR 3.x 返回 OCRResult 对象
ocr_data = result[0]
rec_texts = ocr_data.rec_texts    # ['A', '7', 'k', '9']
rec_scores = ocr_data.rec_scores  # [0.99, 0.98, 0.97, 0.99]
rec_polys = ocr_data.rec_polys    # 边界框坐标列表

# 合并结果
full_text = "".join(rec_texts)    # "A7k9"
```

### 6.1 架构设计

```
OCR 识别结果
     │
     ▼
┌─────────────────────────────────┐
│  投票机制 (Counter Voting)       │
│  4 种策略结果多数投票             │
└─────────────┬───────────────────┘
              │
              ▼
┌─────────────────────────────────┐
│  验证码格式验证                  │
│  - 长度: 3-10 字符              │
│  - 字符合法性: 字母数字          │
└─────────────┬───────────────────┘
              │
              ▼
┌─────────────────────────────────┐
│  候选字符生成                    │
│  - 混淆矩阵单字符替换            │
│  - 常见模式替换 (rn→m, vv→w)     │
│  - 首字符前缀添加 (处理丢失)     │
│  - 子串提取 (处理过长)           │
└─────────────┬───────────────────┘
              │
              ▼
┌─────────────────────────────────┐
│  相似度评分                      │
│  Score = 0.4 × 编辑距离得分      │
│        + 0.6 × 字符匹配得分      │
└─────────────┬───────────────────┘
              │
              ▼
       最优候选结果
```

### 6.2 字符混淆矩阵

| 易混淆组 | 字符 | 混淆模式 |
|---------|------|---------|
| 数字 0 | 0, O, o, Q | 圆形字符 |
| 数字 1 | 1, I, l, i, L | 竖线字符 |
| 数字 2 | 2, Z, z | Z 形字符 |
| 数字 5 | 5, S, s | S 形字符 |
| 数字 6 | 6, b, G | 圆+竖线 |
| 数字 8 | 8, B | 双圆 |
| 数字 9 | 9, g, q | 圆+尾 |
| 数字 7 | 7, T, t | 横+竖 |
| 字母 w | w, vv, VV | V 组合 |
| 字母 m | m, rn, nn | n 组合 |

### 6.3 Levenshtein 编辑距离

用于计算两个字符串的相似度，考虑三种操作：
- **插入**: 添加一个字符
- **删除**: 删除一个字符
- **替换**: 替换一个字符

示例:
```
"qxj" → "6qxj": 距离 = 1 (插入 "6")
"bs8L" → "78sq": 距离 = 4 (替换所有字符)
```

### 6.4 评分算法

```python
score = (similarity * 0.4) + (char_match_score * 0.6)

similarity = 1.0 - (edit_distance / max_length)
char_match_score = similar_chars_count / max_length
```

权重说明:
- **编辑距离 (40%)**: 衡量整体差异
- **字符匹配 (60%)**: 衡量字符混淆程度

### 6.5 实际效果

| 输入 | 输出 | 评分 | 结果 |
|------|------|------|------|
| "2zrw" | "2zrw" | 1.0 | ✓ 正确 |
| "QxJ" | "6qxj" | 0.85 | ✓ 纠正成功 |
| "zwa" | "azwa" | 0.80 | ✓ 纠正成功 |
| "bs8L" | "78sq" | 0.30 | ✗ 无法纠正 |
